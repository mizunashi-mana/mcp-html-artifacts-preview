# 開発ワークフロー

## 基本方針

- **対話的に進める**: ユーザーと確認しながら段階的に実装する
- **早期フィードバック**: 小さな単位で動作確認し、問題を早期に発見する
- **小さな変更の積み重ね**: 大きな変更は分割して段階的にコミットする

## ブランチ戦略

GitHub Flow を採用する。

- `main`: 常にリリース可能な状態を維持
- フィーチャーブランチ: `main` から分岐し、PR でマージ

### ブランチ命名規則

- `feat/{name}`: 新機能
- `fix/{name}`: バグ修正
- `refactor/{name}`: リファクタリング
- `docs/{name}`: ドキュメント
- `chore/{name}`: その他

## タスク実行フロー

### 着手前

1. `.ai-agent/tasks/YYYYMMDD-{name}/README.md` にタスクの目的・ゴール・方針を記述
2. 関連する steering ドキュメント（product.md、tech.md）を確認
3. ユーザーに方針を提示して確認

### 実装中

1. ブランチを作成（`feat/{name}` など）
2. 小さな単位でコミット
3. 品質チェックを実施:
   - `npm run lint` でリントエラーがないことを確認
   - `npm run test` でテストが通ることを確認
   - `npm run build` でビルドが成功することを確認

### 完了時

1. `/autodev-create-pr` でプルリクエストを作成
2. タスク README に完了を記録
